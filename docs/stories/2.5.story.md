## Descricao

Implementar autenticacao simples para proteger o Content Studio. A gestora de marketing deve fazer login com credenciais fixas. Nao e necessario sistema de usuarios completo — apenas proteger o acesso ao sistema.

## Acceptance Criteria

- [ ] Tela de login no frontend com usuario e senha
- [ ] Endpoint `/api/auth/login` que valida credenciais e retorna token JWT
- [ ] Middleware de autenticacao no FastAPI para rotas protegidas
- [ ] Endpoints publicos: `/api/health`, `/api/auth/login`
- [ ] Todos os demais endpoints requerem header `Authorization: Bearer {token}`
- [ ] Token expira em 24 horas
- [ ] Frontend salva token no localStorage e envia em todas as requests
- [ ] Credenciais configuradas via variavel de ambiente (STUDIO_USER, STUDIO_PASS)
- [ ] Redirect para login quando token expirado ou ausente

## Tasks

1. Instalar `python-jose[cryptography]` para JWT
2. Criar `auth.py` com logica de autenticacao
3. Adicionar endpoint `/api/auth/login`
4. Criar middleware de validacao JWT
5. Atualizar frontend com tela de login
6. Adicionar header Authorization em todas as chamadas fetch
7. Implementar logout e redirect em caso de 401
