## Descricao

Configurar GitHub Actions para CI/CD basico: testes automaticos em PRs, deploy automatico do frontend no cPanel e trigger de deploy da API no Render. Base para o CRON semanal futuro (Story 4.7).

## Acceptance Criteria

- [ ] Workflow `ci.yml`: roda testes Python em cada PR
- [ ] Workflow `deploy-frontend.yml`: upload via cPanel UAPI em push para main
- [ ] Workflow `deploy-api.yml`: trigger deploy no Render via webhook
- [ ] Secrets configurados no GitHub: CPANEL_USER, CPANEL_TOKEN, RENDER_HOOK_URL
- [ ] Badge de status no README
- [ ] Testes passando no CI
- [ ] Deploy automatico funcional apos merge em main

## Tasks

1. Criar `.github/workflows/ci.yml`
2. Criar `.github/workflows/deploy-frontend.yml`
3. Criar `.github/workflows/deploy-api.yml`
4. Configurar secrets no GitHub repo
5. Testar pipeline completo (PR → merge → deploy)
6. Adicionar badge no README
