## Descricao

Implementar o modulo SEO/Blog no Content Pipeline conforme arquitetura definida em 15.3. O modulo deve gerar artigos otimizados, landing pages de produto e schema markup automaticamente.

### Componentes a implementar

1. **Blog Post Generator** — Gera artigos SEO a partir de pillar content + claims
2. **Landing Page Generator** — Paginas de produto com specs, depoimentos, CTAs
3. **Schema Generator** — Gera JSON-LD para MedicalDevice, Product, FAQ
4. **Keyword Tracker** — Monitora posicoes e sugere otimizacoes
5. **Sitemap Generator** — Sitemap XML automatico

### Endpoints planejados

| Endpoint | Metodo | Descricao |
|----------|--------|-----------|
| `/api/seo/articles` | GET/POST | Listar/gerar artigos SEO |
| `/api/seo/landing-pages` | GET/POST | Listar/gerar landing pages |
| `/api/seo/schema` | GET | Schema markup por produto |
| `/api/seo/keywords` | GET/PUT | Banco de keywords |
| `/api/seo/sitemap` | GET | Sitemap XML |

## Acceptance Criteria

- [ ] Blog Post Generator criando artigos com titulo, meta description, H1-H3, internal links
- [ ] Landing Page Generator com template por produto (specs + claims + CTA)
- [ ] Schema Generator gerando JSON-LD valido (testado no Rich Results Test)
- [ ] Keyword Tracker com banco inicial de 50+ termos
- [ ] Sitemap Generator automatico
- [ ] 5 endpoints REST funcionais
- [ ] UI no Content Studio para gerenciar artigos e keywords
- [ ] Integrado com claims-bank e copywriters para tom de voz correto
- [ ] 2 artigos-modelo gerados para validacao (1 por produto Salk)
