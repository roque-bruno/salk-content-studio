## Descricao

Implementar o conceito de "Design System como cabresto" — os brandbooks YAML nao sao apenas referencia, mas sim constraints inviolaveis aplicados automaticamente em todos os pontos do pipeline. Cada agente/worker que gera conteudo deve carregar e respeitar o brandbook da marca ativamente.

## Acceptance Criteria

- [ ] Modulo `brand_enforcer.py` criado com validacao de marca
- [ ] Funcao `validate_against_brand(content, brand)` retorna violacoes
- [ ] Validacao de cores: hex codes usados devem estar na paleta da marca
- [ ] Validacao de tom: keywords proibidas por marca detectadas
- [ ] Validacao de logo: regra Mendel vs Salk aplicada
- [ ] Shield (compliance) integrado com brand enforcer
- [ ] Prompt builder injeta constraints do brandbook automaticamente
- [ ] Endpoint `/api/compliance/check` usa brand enforcer alem de prohibited terms
- [ ] Testes unitarios para cada tipo de validacao

## Notas Tecnicas

- Cabresto = constraint, nao sugestao. Se viola brandbook, BLOQUEIA
- Exemplo: se copy usa tom tecnico para Salk → violacao
- Exemplo: se prompt NB2 pede glow/dispersao para LEV → violacao
- Brand enforcer deve ser reutilizavel por qualquer modulo do pipeline
