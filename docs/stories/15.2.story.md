## Descricao

Implementar o modulo WhatsApp Business no Content Pipeline conforme arquitetura definida em 15.1. O modulo deve gerar sequencias de nurturing, catalogo de produtos e templates de mensagem aprovados pela Meta.

### Componentes a implementar

1. **WhatsApp Publisher** — Modulo que envia conteudo via WhatsApp Business API
2. **Template Manager** — CRUD de templates HSM com validacao de formato Meta
3. **Sequence Engine** — Motor de sequencias de nurturing por jornada + persona
4. **Catalog Generator** — Gerador de catalogo de produtos com specs e claims
5. **Webhook Receiver** — Receptor de status de entrega e respostas

### Endpoints planejados

| Endpoint | Metodo | Descricao |
|----------|--------|-----------|
| `/api/whatsapp/templates` | GET/POST | Listar/criar templates |
| `/api/whatsapp/sequences` | GET/POST | Listar/criar sequencias |
| `/api/whatsapp/send` | POST | Enviar mensagem/sequencia |
| `/api/whatsapp/catalog` | GET | Catalogo de produtos |
| `/api/whatsapp/webhook` | POST | Webhook Meta (delivery, read) |
| `/api/whatsapp/metrics` | GET | Metricas de engajamento |

## Acceptance Criteria

- [ ] WhatsApp Publisher implementado com envio via Meta Business API
- [ ] Template Manager com CRUD completo e validacao HSM
- [ ] Sequence Engine com 7 fluxos por fase da jornada (minimo Salk)
- [ ] Catalog Generator gerando catalogo KRATUS, LEV, OSTUS, KRONUS
- [ ] Webhook receiver processando status de entrega
- [ ] 6 endpoints REST funcionais e documentados
- [ ] Testes unitarios para cada componente (cobertura > 80%)
- [ ] UI no Content Studio para gerenciar templates e sequencias
- [ ] Integrado com atomizacao (conteudo master → variante WhatsApp)
