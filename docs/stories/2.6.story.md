## Descricao

Criar 4 arquivos YAML de brandbook — um por marca (Salk, Mendel, Manager Grupo, Dayho) — que servem como knowledge base inviolavel para todos os agentes e workers do pipeline. Cada brandbook define tom de voz, paleta de cores, regras visuais, personas-alvo, e restricoes especificas.

## Acceptance Criteria

- [ ] `brandbooks/salk.yaml` criado: tom comercial, cores Salk, personas comerciais, regras visuais
- [ ] `brandbooks/mendel.yaml` criado: tom tecnico/fabricante, cores Mendel, personas tecnicas
- [ ] `brandbooks/manager-grupo.yaml` criado: tom institucional, cores corporativas, posicionamento grupo
- [ ] `brandbooks/dayho.yaml` criado: tom industrial, cores industriais, posicionamento maquinas
- [ ] Cada brandbook inclui: tone_of_voice, color_palette, typography, visual_rules, target_personas, prohibited_terms, logo_usage, content_pillars
- [ ] Endpoint `/api/data/brandbook/{brand}` retorna brandbook completo
- [ ] Services.py carrega brandbooks com caching
- [ ] Frontend exibe brandbook na secao de Assets

## Tasks
