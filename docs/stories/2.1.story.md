# Story 2.1 — Separar frontend do backend

**Epico:** 2 — Fundacao e Deploy
**Status:** Done
**Executor:** @dev (Dex)
**PRD:** docs/prd/PRD-content-studio-v3.md

## Descricao

Separar o Content Studio em dois artefatos independentes: (1) frontend estatico puro (HTML/CSS/JS com Alpine.js) e (2) API FastAPI independente. O frontend deve funcionar como SPA estatica sem dependencia de servidor Python para servir arquivos. A API deve aceitar CORS de qualquer origem para permitir deploy separado.

## Acceptance Criteria

- [ ] Frontend extraido para pasta dedicada `packages/content-studio-frontend/`
- [ ] `index.html` atualizado para apontar para API via variavel `API_BASE_URL` configuravel
- [ ] Arquivo `config.js` criado com `API_BASE_URL` que pode ser alterado sem rebuild
- [ ] API FastAPI mantem todos os 41 endpoints funcionais
- [ ] API remove servico de arquivos estaticos do frontend (mantendo apenas `/output` e `/assets`)
- [ ] CORS configurado para aceitar origens especificas (studio.salk.com + localhost)
- [ ] Health check `/api/health` retorna `api_url` para validacao de conectividade
- [ ] Frontend funciona abrindo `index.html` direto no browser (modo dev)
- [ ] Testes existentes continuam passando

## Tasks

1. Criar estrutura `packages/content-studio-frontend/`
2. Mover `index.html` e assets estaticos para nova pasta
3. Criar `config.js` com `API_BASE_URL` configuravel
4. Atualizar todas as chamadas `fetch()` no frontend para usar `API_BASE_URL`
5. Remover servico de `index.html` do FastAPI (manter `/output` e `/assets`)
6. Ajustar CORS para origens especificas
7. Atualizar health check com `api_url`
8. Testar frontend standalone e API independente
