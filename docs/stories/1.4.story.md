# Story 1.4 — Executar pipeline completo com 1 peca LEV (end-to-end)

**Epico:** 1 — Calibracao & Primeiro Batch Real
**Status:** Ready
**Executor:** Squad completo (todos os 8 agentes)
**PRD:** docs/prd/PRD-content-production.md v2.0
**Depende de:** Story 1.3

## Descricao

Executar o pipeline completo de producao v2.0 (8 stages) com 1 peca LEV para Salk Medical, do brief ate a peca pronta para publicacao. Validar cada stage do pipeline, medir tempos, e identificar gargalos.

## Acceptance Criteria

- [ ] Stage 1 (Briefing): Atlas gera brief estruturado para 1 peca LEV
- [ ] Stage 2 (Copy): Helix escreve copy usando claims-bank.yaml (framework PAS ou AIDA)
- [ ] Stage 3 (Review Intermediario): Lens valida copy ANTES do visual — aprovado ou ajustado
- [ ] Stage 4 (Visual): Apex gera VDP completo com prompt NB2 + instrucoes Canva
- [ ] Stage 5 (Atomizacao): Nova gera 5-8 derivados cross-platform
- [ ] Stage 6 (Review Final): Lens executa QC layers 1-2
- [ ] Stage 7 (Compliance): Shield executa gate BLOQUEANTE layers 3-4 — PASS
- [ ] Stage 8 (Publicacao): Peca pronta para publicacao manual pelo usuario
- [ ] Tempo total medido e documentado
- [ ] Zero violacoes de compliance

## Tasks

1. Atlas: criar brief (claim, formato, plataforma, persona, pilar, logo)
2. Helix: escrever copy (headline, body, CTA, hashtags) usando brief + claims-bank
3. Lens: review intermediario do copy (tom, claims, persona, formato)
4. Apex: gerar VDP (prompt NB2 8-dimensoes + instrucoes composicao Canva)
5. Nova: atomizar master em 5-8 derivados (adaptar copy, formato, dimensao por plataforma)
6. Lens: review final (QC layers 1-2: spelling, brand, tone, brief alignment)
7. Shield: compliance gate (QC layers 3-4: claims, ANVISA, legal)
8. Documentar resultado e tempos por stage
