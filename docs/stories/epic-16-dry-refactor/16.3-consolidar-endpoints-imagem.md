## Objetivo

Unificar os 3 endpoints duplicados de geracao de imagem em 1 endpoint canonico com parametros.

## Contexto

O diagnostico identificou 3 endpoints fazendo geracao de imagem com configuracoes ligeiramente diferentes. Bug no NB2 precisou de 5 deploys porque a correcao tinha que ser replicada em 3 lugares.

## Acceptance Criteria

- [ ] Mapear os 3 endpoints de imagem existentes e suas diferencas
- [ ] Criar 1 endpoint unificado `/api/production/generate-image` com parametros para cobrir todos os casos
- [ ] Criar aliases/redirects dos endpoints antigos para o novo (retrocompatibilidade)
- [ ] Logica NB2 vs FLUX centralizada em 1 unico ponto de decisao
- [ ] Usar constantes de 16.1 e utils de 16.2
- [ ] Frontend continua funcionando sem alteracao (mesmas responses)
- [ ] Zero regressao — NB2 /edit e /generate funcionam identico
