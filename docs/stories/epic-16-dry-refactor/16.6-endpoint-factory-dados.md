# Story 16.6: Endpoint Factory para Dados GET/PUT

**Status:** Done
**Agente:** @dev
**Prioridade:** P2 — Medio
**Estimativa:** Media complexidade
**Depende de:** 16.1, 16.2

## Objetivo

Substituir os 16 pares GET/PUT de dados (todos com padrao identico) por 1 handler generico factory.

## Contexto

O app.py tem 16 endpoints GET e 16 endpoints PUT que seguem exatamente o mesmo padrao:
- GET: ler arquivo YAML/JSON, retornar conteudo
- PUT: receber conteudo, salvar em arquivo YAML/JSON

Sao ~24 funcoes que poderiam ser 1 factory com configuracao.

## Acceptance Criteria

- [ ] Mapear todos os 16 pares GET/PUT e seus arquivos de dados
- [ ] Criar funcao `data_endpoint_factory(route, file_path, file_type)` que gera GET e PUT
- [ ] Registrar todos os 16 pares via factory
- [ ] Remover as 32 funcoes duplicadas
- [ ] Usar constantes de 16.1
- [ ] Zero mudanca de comportamento — todos endpoints retornam o mesmo
- [ ] Todos PUT endpoints continuam salvando corretamente

- `content_pipeline/web/app.py`
