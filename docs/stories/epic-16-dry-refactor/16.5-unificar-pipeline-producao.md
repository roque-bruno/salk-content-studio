# Story 16.5: Unificar Pipeline de Producao

**Status:** Deferred (pipelines servem use cases diferentes, logica core ja esta nos services)
**Agente:** @dev
**Prioridade:** P1 — Critico
**Estimativa:** Alta complexidade
**Depende de:** 16.1, 16.2, 16.3, 16.4

## Objetivo

Unificar os pipelines de producao que compartilham logica: `produce_single_piece` (L2299), `produce_week` (L1552), e `auto_fill_calendar` (L1545).

## Contexto

`_produce_slot` nao existe — a auditoria real identificou 3 funcoes de producao:
- `produce_single_piece` (L2299): pipeline completo para 1 peca (briefing→copy→prompt)
- `auto_fill_calendar` (L1545): preenchimento bulk de slots do calendario
- `produce_week` (L1552): orquestracao de producao semanal

Compartilham logica significativa de briefing/copy/prompt generation.

## Acceptance Criteria

- [ ] Analisar diferencas exatas entre produce_single_piece, auto_fill_calendar e produce_week
- [ ] Criar funcao core `_produce_piece_core()` com a logica compartilhada
- [ ] `produce_single_piece`, `auto_fill_calendar` e `produce_week` passam a ser wrappers finos que chamam o core
- [ ] Usar endpoints unificados de 16.3 e 16.4
- [ ] Usar constantes de 16.1 e utils de 16.2
- [ ] Zero mudanca de comportamento — producao manual e automatica funcionam identico
- [ ] Testar producao de peca individual
- [ ] Testar producao batch/semana
