# Story 16.7: Consolidar Funcoes Frontend

**Status:** Done
**Agente:** @dev
**Prioridade:** P1 — Critico
**Estimativa:** Alta complexidade
**Depende de:** 16.1, 16.2

## Objetivo

Eliminar duplicacoes no frontend JavaScript (index.html): wrappers de imagem, criacao de peca, batch operations, e padronizar tratamento de erro.

## Contexto

O diagnostico identificou:
- 5 funcoes wrapper de geracao de imagem (quase identicas)
- 3 caminhos diferentes para criacao de peca
- 4 loops de batch identicos (copy-paste)
- 2 implementacoes de batchGenerateCopy (uma sem confirmacao)
- 115 try-catch com 3 padroes diferentes
- loadPieces() chamada 13+ vezes apos cada micro-operacao

## Acceptance Criteria

- [ ] Unificar 5 wrappers de imagem em 1 funcao `generateImage(pieceId, options)`
- [ ] Unificar 3 caminhos de criacao de peca em 1 funcao `createPiece(data)`
- [ ] Unificar 4 loops batch em 1 funcao `batchOperation(pieceIds, operation, options)`
- [ ] Remover batchGenerateCopy duplicada (manter a que tem confirmacao)
- [ ] Padronizar try-catch: 1 funcao helper `apiCall(fn, errorMsg)` que wrapa todas as chamadas
- [ ] Otimizar loadPieces(): debounce ou consolidar chamadas consecutivas
- [ ] Zero mudanca visual ou de comportamento
- [ ] Todos botoes do frontend funcionam identico
