## Descricao

Analise de dados do Journey Log para otimizar o pipeline automaticamente. Coleta taxas de sucesso/falha por fase e agente, identifica gargalos e ajusta roteamento de modelos LLM com base em performance historica.

## Acceptance Criteria

- [ ] Coleta de metricas do Journey Log (sucesso/falha por fase)
- [ ] Analise de performance por agente do pipeline
- [ ] Identificacao automatica de gargalos e padroes de falha
- [ ] Sugestoes de otimizacao baseadas em dados historicos
- [ ] Auto-ajuste de roteamento de modelos LLM por tipo de tarefa
- [ ] Dashboard ou relatorio de metricas acessivel
- [ ] Testes unitarios cobrindo analise e sugestoes

## Tasks

1. Implementar parser de Journey Log
2. Implementar coleta de metricas por fase e agente
3. Implementar analise de taxas de sucesso/falha
4. Implementar deteccao de gargalos
5. Implementar sugestoes de otimizacao
6. Implementar auto-ajuste de roteamento de modelos
7. Criar relatorio/dashboard de metricas
8. Escrever testes unitarios

## Dependencias

- Stories 4.1-4.6 (pipeline completo com Journey Log ativo)
- Volume de dados suficiente no Journey Log para analise significativa
